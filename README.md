# only-ai-admin

这个仓库用于记录和沉淀「AI 原生框架设计规则」的思想。

我们当前讨论的目标不是先实现某一种编程语言的 admin 框架，而是先制定一套更上层的规则、原则和标准：什么样的框架才适合 AI Agent 理解、操作、验证和长期演进。

后续可以基于这套思想，再为不同开发语言、不同技术栈建立独立仓库，实现具体的 AI 原生 admin 框架。

## 当前定位

**only-ai-admin 是 AI 原生 admin 框架的思想与标准仓库。**

它关注：

- AI 原生框架应该遵守什么设计原则
- 框架如何把隐式知识显式化
- 框架如何面向 AI Agent 暴露自描述能力
- 框架如何沉淀模块、功能、接口、文档和决策
- 框架如何支持长期演进，而不只是一次性代码生成

## 重要区分

本仓库不直接绑定具体语言或具体 Web 框架。

它不是：

- 某个 TypeScript/Python/Go/Rust admin 框架
- 某个前端组件库
- 某个后端脚手架
- 某个业务项目模板

它是：

- AI 原生框架设计准则
- AI 友好框架标准
- 后续语言级 admin 框架的思想源头

## 后续仓库关系

未来可以基于本仓库建立多个具体实现仓库，例如：

- `only-ai-admin-ts`
- `only-ai-admin-python`
- `only-ai-admin-go`
- `only-ai-admin-java`
- `only-ai-admin-rust`

这些仓库负责具体语言和技术栈的实现；本仓库负责标准、原则、概念和范式沉淀。
